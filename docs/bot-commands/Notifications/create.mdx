This command allows you to create notifications in several notifications categories

```
/notification create
```

## NFT

<Tabs groupId="platforms">
<TabItem value="discord" label="Discord" default>

```
/notification create nft
```

<p>
  <strong>Parameters:</strong>
</p>

\*_Required_

**blockchain\*:**

    type: Option
    description: Blockchain where that address exists.
    choices: [ Terra Classic, Terra ]

**collection\*:**

    type: Option
    description: NFT collection address to subscribe.
    choices: [ NFT Collections supported on our Assets ]

**token:**

    type: Option
    description: If this parameter is filled, you create an NFT Monitor for a specific NFT.
    choices: [ All NFT tokens from the collection previously selected ]

**channel:**

    type: Channel
    description: Channel to send the alerts.

:::info
After running the command you will be prompted with a message with some alert types to choose from, please press the ones you wish and if the button turns green your option is recorded.
:::

**Example Usage**

<Tabs>
  <TabItem value="nftMonitor" label="NFT Monitor on DMs" default>
    <CodeBlock>
      /notification create nft blockchain:Terra collection:Skeleton Punks
      token:Skeleton Punk #878 (878)
    </CodeBlock>
    <Admonition type="info">
      <p>You must execute this command in the bot's DMs.</p>
    </Admonition>
  </TabItem>
  <TabItem value="nftMonitorServer" label="NFT Monitor on Server">
    <CodeBlock>
      /notification create nft blockchain:Terra collection:Skeleton Punks
      token:Skeleton Punk #878 (878) channel:#nft-channel
    </CodeBlock>
  </TabItem>
  <TabItem value="nftSales" label="NFT Sales">
    <CodeBlock>
      /notification create nft blockchain:Terra collection:Skeleton Punks
      channel:#sales-channel
    </CodeBlock>
  </TabItem>
</Tabs>

## Governance

<Tabs groupId="platforms">
<TabItem value="discord" label="Discord" default>

```
/notification create governance
```

<p>
  <strong>Parameters:</strong>
</p>

\*_Required_

**blockchain\*:**

    type: Option
    description: Blockchain where the poll exists.
    choices: [ Terra Classic, Terra ]

**project\*:**

    type: Option
    description: Select the project from which you want to receive governance notifications.

**channel\*:**

    type: Channel
    description: Channel where the polls will be sent to.

:::note
In order for a governance polls to be accepted they must follow the standard used by Anchor Protocol and Mirror Protocol.
:::

**url:**

    type: URL
    description: Insert the protocol website where all the proposals https://protocol.io/gov/%id%.

**mention:**

    type : Role
    description: Insert which role you wish to mention whenever a new poll is posted.

**Usage example**

```
/notification create governance blockchain:Terra channel:#governance address:terra...09 url:https://protocol.io/gov/%id% mention:@role
```

## Domain Changes

<Tabs groupId="platforms">
<TabItem value="discord" label="Discord" default>

```
/notification create domain-changes
```

<p>
  <strong>Parameters:</strong>
</p>

\*_Required_

**blockchain:\***

    type: Option
    description: Blockchain where that domain exists.
    choices: [ Terra Classic, Terra ]

**domain:**

    type: String | Regex
    description: The domain you want to be alerted about | Regex to match single or multiple domains.

:::info

- Keep in mind that the domain must be in the format of `domain.tld` and not just the name of the domain.
- If you provide a regex you must set the `regex` parameter to true.
- If you want to match multiple domains you can use the regex `.*` to match any character and `|` to separate the domains.

:::

**regex:**

    type: Boolean
    description: Whether to use regex or not in the domain.
    choices: [ True or False ]

**channel:**

    type: Channel
    description: Channel where the domain changes alerts will be sent to

:::info
After running the command you will be prompted with a new message with some alert types to choose from, please press the ones you wish and if the button turns green your option is recorded.
:::

**Example Usage**

```
/notification create domain-changes blockchain: Terra domain: Hermes Protocol channel: #domain feed
```
